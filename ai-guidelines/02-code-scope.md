# ✏️ 02. Code Scope Rules (코드 작성/수정 범위 규칙)

AI는 코드 작성 및 리팩토링 시 **허용된 영역**과 **금지된 영역**을 반드시 구분해야 한다.  
구체적인 기술별 규칙(Next.js, Spring Boot)은 각 `conventions/` 디렉토리에 정의되어 있으며,  
이 문서는 **AI가 따라야 하는 기본 상위 원칙**을 제시한다.

---

## 📌 주석 태그 규칙

코드 파일 내부에는 AI가 따를 수 있는 주석 태그가 포함될 수 있다.

- `AI:DO-NOT-TOUCH`  
  → 절대 수정 금지. (보안 로직, 핵심 알고리즘, 민감한 Config 포함)

- `AI:EDIT-ALLOWED`  
  → 가벼운 리팩토링/포맷팅 허용. (예: 변수명 개선, 주석 추가)

- `AI:CHANGE-ZONE:start` ~ `AI:CHANGE-ZONE:end`  
  → 이 블록 안에서는 AI가 자유롭게 코드 생성/수정 가능.  
  → 새 기능 구현 시 기본적으로 이 영역을 활용.

- `AI:SENSITIVE`  
  → 보안/민감정보 관련 코드. 읽기 전용, 절대 수정 금지. (예: JWT, 암호화)

---

## 📌 허용되는 작업
- 새로운 기능 추가: `CHANGE-ZONE` 내부에만 추가  
- 리팩토링: `EDIT-ALLOWED` 태그가 붙은 범위 내  
- 문서화/주석 추가: 어디든 가능 (단, `DO-NOT-TOUCH`/`SENSITIVE` 영역 제외)  
- 테스트 코드 작성: 각 기술별 테스트 경로에 생성 가능  
  - Next.js → `__tests__/`, `src/tests/`  
  - Spring Boot → `src/test/java/...`  

---

## 📌 금지되는 작업
- `DO-NOT-TOUCH` 또는 `SENSITIVE` 태그가 붙은 코드 수정  
- 환경 변수/시크릿(`.env`, `application.yml`) 직접 수정  
- DB 마이그레이션 스크립트(`db/migration/`) 직접 생성/삭제  
- 배포/CI 스크립트(`.github/workflows/`, `scripts/`) 변경  

---

## 📌 태그가 없는 경우
- 태그가 없는 영역은 **기본적으로 읽기 전용**으로 간주한다.  
- 단, 수정이 반드시 필요하다고 판단되는 경우:  
  1. 리뷰어(팀원)에게 해당 영역에 **적절한 태그를 추가해달라고 요청**한다.  
  2. 혹은 리뷰어에게 **직접 허용 여부를 확인받은 후** 수정한다.  
- 승인 없이 태그 없는 코드를 무단 수정하는 것은 금지된다.  

---

## ✅ 원칙
1. AI는 **항상 주석 태그를 먼저 확인**한다.  
2. 태그가 없는 영역은 **읽기 전용**이지만, 필요 시 반드시 리뷰어에게 허용 여부를 확인한다.  
3. 기능 확장은 `CHANGE-ZONE`에, 리팩토링은 `EDIT-ALLOWED`에 한정한다.  
4. 기술별 세부 규칙은 **`conventions/02-frontend/`** 또는 **`conventions/02-backend/`** 문서를 따른다.
