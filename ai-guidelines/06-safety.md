# 🔒 06. Safety & Sensitive Information Rules (보안/민감정보 규칙)

AI는 코드 작성 및 문서화 과정에서 **보안과 민감정보 보호**를 최우선으로 한다.  
비밀번호, 토큰, 인증서 등은 절대 코드나 문서에 직접 포함되지 않아야 한다.  

---

## 📌 민감정보 관리
- 환경 변수 파일(`.env`, `.env.local`, `application.yml`, `application-dev.yml`, `application-prod.yml`)은 절대 수정하지 않는다.  
- API 키, DB 접속 정보, OAuth 토큰은 코드/문서에 노출하지 않고 반드시 **환경 변수**로 관리한다.  
- GitHub Actions, CI/CD 시크릿 값은 수정하거나 출력하지 않는다.  

---

## 📌 로그/출력 제한
- 콘솔, 로그 파일, 테스트 출력에 **민감정보(토큰, 비밀번호, 주민번호 등)**를 절대 남기지 않는다.  
- 이는 **디버깅 목적**이라도 예외가 없다.  
  - 예시 (❌ 금지):
      ```java
  
      log.info("User password={}", user.getPassword()); // 금지
  
      ```
  - 필요 시, **마스킹 처리** 후 출력한다:
      ```java
  
      log.info("User password hash prefix={}", passwordHash.substring(0, 5));
  
      ```

---

## 📌 외부 연동 주의사항
- 외부 API 호출 시, 반드시 **환경 변수에 정의된 엔드포인트/키**를 사용한다.  
- 테스트/문서 예시에서는 **가짜 값**(`FAKE_API_KEY`, `dummy@example.com`)을 사용한다.  

---

## 📌 금지되는 작업
- GitHub, Slack, Notion 등 외부 툴에 **실제 비밀 값**을 노출하는 것  
- DB 마이그레이션 파일에 **샘플 데이터로 민감정보**를 포함하는 것  
- Commit/PR 메시지, 이슈/코멘트에 비밀번호, 토큰, 시크릿 값 기록하는 것  
- ⚠️ **보안 위반이나 민감정보 노출 사례는 반드시 회고록에 기록하고 재발 방지 대책을 마련한다.**

---

## 📌 보안 태그
- `AI:SENSITIVE` 주석이 있는 코드는 AI가 수정하지 않는다.  
- 보안 관련 로직(JWT 인증, 암호화, 비밀번호 해싱 등)은 기본적으로 **DO-NOT-TOUCH** 규칙을 따른다.  
- `AI:SENSITIVE`가 붙은 코드는 **자동으로 DO-NOT-TOUCH**로 간주한다.  

---

## ✅ 원칙
1. 모든 민감정보는 환경 변수나 시크릿 저장소를 통해 관리한다.  
2. 로그, 출력, 문서화 시 절대 실제 민감정보를 포함하지 않는다.  
3. 테스트/예시에서는 반드시 가짜 데이터를 사용한다.  
4. 보안 태그(`AI:SENSITIVE`, `AI:DO-NOT-TOUCH`)가 있는 영역은 절대 수정하지 않는다.  
5. 문서화/커밋/PR 등 외부 협업 툴에도 민감정보를 기록하지 않는다.
